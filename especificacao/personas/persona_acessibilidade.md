# Persona 1: Marcos (Profissional de Acessibilidade)

## 1. Perfil
*   **Nome:** Marcos Oliveira
*   **Idade:** 45 anos
*   **Cargo:** Técnico Administrativo na UFAM
*   **Condição:** Baixa visão (utiliza alto contraste e leitores de tela)

## 2. Objetivos e Necessidades
Marcos é responsável por auditar a conformidade dos projetos acadêmicos com as normas da universidade. Ele precisa acessar dados de múltiplos projetos rapidamente, mas sofre com sistemas que possuem interfaces poluídas ou cores sem contraste.

## 3. Cenário de Uso
Marcos recebe uma solicitação para verificar se todos os bolsistas de um determinado laboratório entregaram seus relatórios parciais. Ele abre o E-Project no seu navegador, ativa o modo de alto contraste e utiliza o teclado para navegar entre as listas de projetos.

## 4. Jornada do Usuário
1.  **Entrada:** Marcos faz login no E-Project.
2.  **Ação:** Ativa o botão global de "Alto Contraste" no cabeçalho.
3.  **Busca:** Utiliza a barra de busca para filtrar pelo nome do laboratório.
4.  **Análise:** Navega pelas abas de "Documentos" de cada projeto usando a tecla `Tab`.
5.  **Resultado:** Identifica visualmente (com o auxílio do contraste) quais documentos estão com status "Pendente" e exporta um resumo em texto simples para seu relatório de auditoria.
